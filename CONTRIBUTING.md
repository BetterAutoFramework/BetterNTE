# Contributing

Thank you for your interest in contributing to BetterNTE!

## Getting Started

1. Fork the repository.
2. Clone your fork locally.
3. Create a new branch for your feature or bug fix.
4. Make your changes.
5. Submit a pull request.

## Development Environment

Please refer to the development documentation:
- [Development Guide (中文)](docs/development.md)
- [Development Guide (English)](docs/development_EN.md)

## Reporting Issues

- Use GitHub Issues to report bugs.
- Include steps to reproduce, expected behavior, and actual behavior.
- Attach screenshots or logs if applicable.

## Pull Requests

- Keep PRs focused on a single change.
- Write clear commit messages.
- Ensure your code compiles without errors.
- Add tests where appropriate.

## Code Style

- Rust: Follow standard `rustfmt` formatting.
- TypeScript/React: Follow the existing ESLint configuration.
- Use meaningful variable and function names.
